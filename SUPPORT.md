# Support

Socialize is free to use and free to self-host. Sponsorship is voluntary
support for maintenance; it does not purchase product features, priority support,
access to user data, or roadmap control.

## Choose the right channel

| Need | Where to ask |
| --- | --- |
| Reproducible bug or documentation gap | [GitHub issue forms](https://github.com/Llewellyn500/socialize/issues/new/choose) |
| Security vulnerability or exposed credential | [security@socialize.you](mailto:security@socialize.you) |
| Harmful hosted profile, phishing, malware, or impersonation | [safety@socialize.you](mailto:safety@socialize.you) or the profile report route |
| Hosted account, publishing, or handle help | [support@socialize.you](mailto:support@socialize.you) |
| Privacy request | [privacy@socialize.you](mailto:privacy@socialize.you) |
| Sponsorship | [GitHub Sponsors](https://github.com/sponsors/Llewellyn500) or [sponsors@socialize.you](mailto:sponsors@socialize.you) |

## Before you contact us

- Never send passwords, recovery codes, access tokens, private keys, or Firebase
  service-account credentials.
- Include the relevant route, public profile URL, expected result, actual result,
  and steps already tried.
- For a self-hosted deployment, include the template version and deployment
  details, but remove secrets and personally identifiable data first.

Support is handled as time permits. The project does not offer a paid support
queue or service-level agreement.
